02. 제안 배경

2.1 사회적 배경 — 온라인 갈등 심화

2025년 기준 국내 소셜 미디어 이용자는 약 4,600만 명 이상으로, 일상적인 의견 충돌과 온라인 논쟁은 커뮤니티, SNS, 메신저를 가리지 않고 폭발적으로 증가하고 있습니다.

  • 커뮤니티 양극화: 정치, 젠더, 세대 갈등 등 주제별 진영 논리가 강화되며 건설적 토론이 실종되고 있음
  • 감정적 소비: 논쟁이 '승리'가 아닌 '분노 해소' 목적으로 전환되며 인신공격과 혐오 표현 증가
  • 중재 부재: 기존 온라인 공간에는 객관적 판정 시스템이 존재하지 않아, 논쟁이 평행선을 달리거나 감정 싸움으로 변질
결론: 온라인 논쟁에 대한 객관적이고 공정한 판결 시스템에 대한 사회적 수요가 존재하며, 이를 해결할 수 있는 기술적 기반이 성숙해졌음.

2.2 기술적 배경 — 생성형 AI 성숙

  • LLM 성능 고도화: GPT-4.1, Gemini 2.5, Claude Sonnet 4 등 최신 모델들이 자연어 분석, 논리 평가, 텍스트 생성에서 인간 수준에 근접한 성능 달성
  • 멀티모델 크로스 체크: 단일 AI의 편향(hallucination) 리스크를 3사 독립 판결 + 다수결 방식으로 구조적으로 완화 가능
  • API 접근성 대중화: OpenAI, Google, Anthropic 모두 개발자 친화적 API를 제공하며, 판결당 비용이 ~185원 수준(GPT-4.1 80원 + Claude Sonnet 4 100원 + Gemini 5원)으로 서비스화 가능한 단가까지 하락
  • 콘텐츠 안전 기술: AI 기반 유해 콘텐츠 필터링(선동/혐오/불법) 기술이 성숙하여 플랫폼 운영 안전성 확보 가능
  • 실시간 통신 기술: Socket.io 기반 WebSocket 통신이 안정적으로 성숙하여, 실시간 채팅 논쟁 + 프레즌스 시스템 구현이 용이
결론: 기술적으로는 이미 구현 가능하지만, 3개 AI를 독립적으로 병렬 호출하여 멀티 판결을 제공하는 서비스는 아직 존재하지 않는 블루오션 영역임.

2.3 필요성

#문제현황필요성
1 객관적 판정 부재 온라인 논쟁에서 승패를 가릴 수 있는 객관적 기준이 없어 끝없는 평행선 AI 기반 정량적 채점(5항목 루브릭)으로 논쟁에 구조와 결론을 부여
2 단일 관점 편향 1개 AI만 사용 시 모델 고유의 편향이 판결에 반영될 위험 3사 AI 독립 판결 + 다수결로 교차 검증하여 공정성 극대화
3 시민 참여 부족 AI 판결만으로는 사회적 합의를 반영하기 어려움 AI 각 25%(3사 75%) + 시민 25% 복합 판결로 기계적 판단과 인간적 판단의 균형
4 유해 콘텐츠 확산 논쟁 플랫폼이 혐오/선동 발언의 온상이 될 위험 3단계 콘텐츠 방어 시스템(비속어+AI필터+게이트키핑)으로 건전한 논쟁 환경 보장
5 AI 시대 논리적 사고력 필요 생성형 AI 시대에 프롬프트 작성 능력과 논리적 사고력이 핵심 역량으로 부상했지만, 이를 자연스럽게 훈련할 수 있는 플랫폼이 부재 논쟁 주장을 작성하는 과정 자체가 자신의 생각을 구조화하고 근거를 정리하는 훈련이 되며, AI가 5항목 루브릭으로 즉각적인 피드백을 제공하여 논리적 사고력을 자연스럽게 향상
6 다양한 관점 이해 부족 온라인 커뮤니티의 확증 편향(echo chamber)으로 자기 진영의 논리만 소비하며, 반대 의견을 이해하려는 시도 자체가 줄어드는 추세 논쟁 과정에서 상대측 주장을 읽고 AI의 양측 분석 판결문을 확인함으로써, 반대 논리의 타당한 지점을 자연스럽게 인식하게 되어 세상을 더 넓은 관점에서 바라보는 시야를 확장
핵심 가치: Moragora는 단순히 누가 이겼는지를 판정하는 서비스가 아닙니다. AI 시대에 필수적인 논리적 사고력을 재미있는 논쟁을 통해 자연스럽게 훈련하고, 상대방의 주장과 AI의 양측 분석을 통해 나와 다른 관점을 이해하는 폭을 넓혀주는 — 놀이이자 성장 도구입니다. 주장을 작성하면서 프롬프트처럼 생각을 구조화하는 능력이 길러지고, 판결문을 읽으면서 "상대의 논리에도 이런 타당한 점이 있구나"를 깨닫게 되는 경험은 어떤 토론 교육보다 효과적입니다.

04. STP 전략

4.1 Segmentation (시장 세분화)

6.2 복합 판결 (AI 각 25% + 시민 25%)

최종 점수 = ( GPT-4.1 점수 × 0.25 ) + ( Gemini 점수 × 0.25 ) + ( Claude 점수 × 0.25 ) + ( 시민 점수 × 0.25 )
각 AI = 5항목 점수 합산 (100점 만점)  |  시민 점수 = (내 측 투표 수 / 전체 투표 수) × 100
투표 기간: AI 판결 완료 후 시민 투표 기간(기본 24시간, VOTE_DURATION_HOURS 환경변수로 조정 가능)이 시작됩니다. AI 결과는 즉시 당사자에게 표시되고, 시민 투표는 마감까지 별도 진행됩니다.
임계값 규칙: 투표 마감 시점에 시민 투표 참여자가 30명 미만일 경우, 시민 점수는 반영되지 않으며 AI 100%로 판결합니다.

10. 성공 기준

아래 시나리오가 중단 없이 완전하게 동작하면 성공으로 판단합니다.

텍스트 모드 (10단계)

  1. 카카오 로그인 — 소셜 로그인으로 계정 생성 및 인증 완료
  2. 3단계 위자드 — 목적(승부/합의/분석) → 렌즈(논리/감정 등) → 주제+카테고리 입력
  3. 초대 링크 생성 — 고유 링크 복사 후 상대방에게 전달
  4. 상대방 참여 — 초대 링크를 통해 논쟁 방 입장
  5. 주장 입력 (콘텐츠 방어) — 양측 50~2,000자 입력. 3단계 필터 정상 동작
  6. AI 3사 5항목 채점 — GPT-4.1, Gemini, Claude Sonnet 4 병렬 호출 및 루브릭 채점
  7. 판결 결과 표시 — AI 판결 즉시 표시 + status 'voting' 전환 + vote_deadline 설정
  8. 시민 투표 — 투표 기간(24h) 동안 공개 피드에서 시민이 A/B 중 선택하여 투표
  9. 복합 판결 반영 — 투표 마감 후 30명 이상 시 AI 각 25%(75%) + 시민 25% 복합 점수 확정, status 'completed' 전환
  10. 랭킹 갱신 — 판결 결과에 따른 사용자 랭킹 업데이트

채팅 모드 (8단계) v2.0 NEW

  1. 로비 입장 — 공개 방 목록 실시간 표시, 참여자 프레즌스 확인
  2. 방 생성 또는 참여 — 사이드(A/B) 선택 후 채팅방 입장
  3. 실시간 채팅 — 15분 서버 타이머 작동, 비속어 실시간 필터링
  4. 상태 자동 전환 — B측 참여 시 waiting→chatting, 타이머 만료 시 chatting→judging
  5. AI 판결 — 채팅 로그 기반 3사 AI 병렬 판결
  6. 판결 결과 + 시민 투표 — 텍스트 모드와 동일한 복합 판결 플로우
  7. 알림 수신 — 판결 완료 알림, 읽음 처리
  8. 이탈 감지 — disconnect 후 2분 내 미복귀 시 자동 판결 정상 동작
