2. 인증 방식 ▼

Supabase Auth 기반 JWT 토큰 인증을 사용합니다.

Request Header

Authorization: Bearer <supabase_access_token>

인증 미들웨어 (requireAuth)

동작설명
토큰 검증Supabase Admin으로 JWT 검증 후 사용자 정보 추출
req.user 설정검증된 사용자 객체를 req.user에 바인딩
실패 시401 Unauthorized 반환
참고: 공개 API(토론 목록, 판결 피드, 랭킹 등)는 인증 없이 접근 가능합니다. 각 엔드포인트의 AUTH / PUBLIC 뱃지를 확인하세요.

인증: Cron API는 CRON_SECRET 헤더 검증을 통해 보호됩니다. 외부 스케줄러(Vercel Cron, cron-job.org 등)에서만 호출해야 합니다.
POST /api/cron/daily-debate SECRET 데일리 논쟁 자동 생성

설명

매일 자동으로 "오늘의 논쟁"을 생성합니다. AI가 시사적이고 흥미로운 주제를 선정하여 토론을 생성합니다.

Request Header

Authorization: Bearer {CRON_SECRET}

Response 201

{
  "id": "uuid",
  "topic": "AI가 생성한 오늘의 논쟁 주제",
  "category": "daily",
  "status": "arguing",
  "created_at": "ISO8601"
}
DELETE /api/cron/daily-debate/:id SECRET 데일리 논쟁 삭제

설명

특정 데일리 논쟁을 삭제합니다. 잘못 생성된 데일리 논쟁 정리용입니다.

Response 200

{
  "message": "데일리 논쟁이 삭제되었습니다."
}

20. Socket.io 이벤트 [NEW] ▼
연결: Socket.io 서버는 Express 서버와 동일한 포트(5000)에서 실행됩니다. 클라이언트는 io(SERVER_URL)로 연결합니다. 인증은 연결 시 auth.token으로 JWT를 전달합니다. 
// 클라이언트 연결 예시
const socket = io(SERVER_URL, {
  auth: { token: supabase_access_token }
});

Payload

{
  "debateId": "uuid"

Payload

대기실에 입장하여 상대방의 접속 상태를 실시간으로 확인합니다.

Payload

Payload

{
  "debateId": "uuid"

EMIT typing CLIENT→SERVER 타이핑 상태 전송

Payload

{
  "debateId": "uuid",
  "isTyping": true
}

상대방에게 타이핑 중임을 알립니다. isTyping: false로 타이핑 종료를 알립니다.

EMIT start-game CLIENT→SERVER 실시간 논쟁 시작

Payload

{
  "debateId": "uuid"
}

양측이 모두 준비되면 실시간 논쟁(채팅)을 시작합니다. 서버에서 타이머가 시작됩니다.

EMIT send-message CLIENT→SERVER 채팅 메시지 전송

Payload

{
  "debateId": "uuid",
  "content": "메시지 내용",
  "side": "A"
}

채팅 메시지를 전송합니다. 서버에서 DB 저장 후 방의 모든 참여자에게 브로드캐스트합니다. 토론 상태가 waiting일 경우 자동으로 chatting으로 전환되며 서버 타이머가 시작됩니다.

EMIT request-time-change CLIENT→SERVER 토론 시간 변경 요청

Payload

{
  "debateId": "uuid",
  "newTime": 600    // 초 단위 (예: 600 = 10분)
}

상대방에게 토론 시간 변경을 제안합니다. 상대방이 수락하면 적용됩니다.

EMIT vote-time-change CLIENT→SERVER 시간 변경 투표 (수락/거절)

Payload

{
  "debateId": "uuid",
  "approved": true    // true: 수락, false: 거절
}

  "reason": "manual"

Payload

{

설정된 토론 시간이 만료되어 채팅이 자동 종료되었을 때 발생합니다.

ON filter-blocked SERVER→CLIENT 메시지 필터 차단 알림

Payload

{
  "reason": "부적절한 표현이 포함되어 있습니다."
}

전송한 메시지가 콘텐츠 필터에 의해 차단되었을 때 발신자에게만 전달됩니다.

ON time-change-request SERVER→CLIENT 시간 변경 요청 수신

Payload

{
  "requestedBy": "uuid",
  "nickname": "닉네임",
  "newTime": 600    // 초 단위
}

상대방이 토론 시간 변경을 요청했을 때 발생합니다. UI에서 수락/거절 버튼을 표시합니다.

ON time-change-approved SERVER→CLIENT 시간 변경 승인

Payload

{
  "newTime": 600,
  "approvedBy": "uuid"
}

시간 변경 요청이 승인되어 새 시간이 적용되었을 때 발생합니다.

ON time-change-cancelled SERVER→CLIENT 시간 변경 거절

Payload

{
  "cancelledBy": "uuid",
  "reason": "rejected"
}

시간 변경 요청이 거절되었을 때 발생합니다.
