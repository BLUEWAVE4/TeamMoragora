### 1.4 생존/소멸을 가르는 5가지 핵심 조건

```
10년 생존 서비스의 공통 DNA
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

① UGC 콘텐츠 엔진    → 사용자가 콘텐츠를 만들어야 소진되지 않음
② 네트워크 효과       → 사용자가 많을수록 가치가 기하급수적으로 증가
③ 사용자 자산 축적    → 떠나면 잃는 것이 있어야 함 (Lock-in)
④ 플랫폼 전환/확장    → 단일 기능 → 다기능 플랫폼으로 진화
⑤ 생활 밀착 유틸리티  → "없으면 불편한" 수준의 실용 가치

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
소멸 서비스의 공통 사인(死因)

✗ 콘텐츠 소진         → 유한한 콘텐츠 = 유한한 수명
✗ 노벨티 의존         → 신기함이 유일한 가치
✗ 동기식 전용         → 실시간에 묶이면 이탈률 급등
✗ 크리에이터/파워유저 무시 → 핵심 기여자가 떠나면 생태계 붕괴
✗ Product로 머물러 Platform이 되지 못함
```

## 2. Verdict 현재 상태 진단: 영속성 관점

### 2.1 5대 생존 조건 대비 현재 수준

| 생존 조건 | 현재 수준 | 진단 |
|-----------|-----------|------|
| ① UGC 콘텐츠 엔진 | **위험** | 사용자가 "실제 분쟁"을 가져와야 콘텐츠가 생성됨. 일상적 분쟁 빈도는 주 1~2회 수준으로, 매일 방문할 이유가 없음. 밸런스 게임 앱의 콘텐츠 소진 패턴과 구조적으로 유사 |
| ② 네트워크 효과 | **취약** | 바이럴 루프(초대/공유)는 있으나, 사용자 수 증가에 따른 서비스 가치 증가 메커니즘이 없음. 소셜 그래프 없음, 댓글/토론 제외(F-034), 팔로우/구독 Q3 이후 |
| ③ 사용자 자산 축적 | **보통** | 점수/랭킹/전적은 있으나 TOP 100 중심으로 85%+ 사용자에게는 무의미. 자유 설정 렌즈와 같은 개인화 자산이 약함 |
| ④ 플랫폼 전환 | **미설계** | MVP가 "도구"(분쟁 입력 → AI 판결)로 설계됨. 플랫폼 전환 계획은 Q2 2027로 12개월+ 후. 그때까지 생존하기 어려울 수 있음 |
| ⑤ 생활 밀착 유틸리티 | **낮음** | "논쟁이 발생했을 때"만 필요. 매일 논쟁이 발생하지 않으므로 일상적 유틸리티가 아님 |

```
          ┌──┐                              ┌──┐
```

### 2.3 5대 영속성 위험 요인 (우선순위순)

| # | 위험 요인 | 심각도 | 설명 |
|---|-----------|--------|------|
| 1 | **콘텐츠 공급 의존성** | Critical | 서비스 내 콘텐츠 생성 엔진 부재. 사용자가 실제 분쟁을 가져와야만 콘텐츠가 생김 → 분쟁 없는 날 = 방문 이유 없음 |
| 2 | **커뮤니티/소셜 레이어 부재** | Critical | 댓글 제외, 팔로우 미설계 → 거래적(transactional) 경험. 입력→결과→이탈 패턴 |
| 3 | **상대방 초대 병목** | High | 60% 예상 이탈 지점. Solo 모드는 랭킹 미반영 → 핵심 리텐션 메커니즘에서 제외 |
| 4 | **AI 비용의 선형 증가** | High | 판결당 240원, 수익화 6~9개월 후. 캐싱/티어링 미설계 |
| 5 | **생태계 기능 지연** | Medium | 토너먼트/크리에이터/소셜은 12~15개월 후 계획 → 노벨티 소멸과 생태계 구축 사이 "몰입 공백" 발생 |

## 3. 영속성 전략: Product → Platform 전환 로드맵

### 3.1 전략 프레임워크: Hook → Habit → Platform

```
현재 (Product)                   목표 (Platform)
━━━━━━━━━━━━━━━━━━━━       ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

"AI가 판결을 내려준다"     →    "논쟁과 판단의 커뮤니티"

분쟁 입력 → AI 판결        →    탐색/투표/토론/생성/경쟁/소비
  (1회성 도구)                    (매일 방문하는 습관)

사용자: 논쟁 당사자         →    논쟁 당사자 + 구경꾼 + 판사 + 큐레이터
  (2명이 필요)                    (혼자도 가치 있음)
```

### 3.2 Phase 1: 콘텐츠 엔진 구축 (MVP~출시 3개월)

> **핵심 원칙**: "내 논쟁이 없어도 매일 방문할 이유를 만든다"

#### 3.2.1 오늘의 논쟁 (Daily Debate)

매일 운영팀/AI가 1개의 논쟁 주제를 제시하고, 모든 사용자가 한쪽을 선택하여 주장을 작성할 수 있는 기능.

```
┌─────────────────────────────────────┐
│  🔥 오늘의 논쟁 #127                │
│                                     │
│  "택배 기사가 문 앞에 놓고 간 택배를  │
│   이웃이 가져간 경우, 책임은?"       │
│                                     │
│  ┌─────────────┐ ┌─────────────┐   │
│  │  A. 택배사   │ │  B. 이웃    │   │
│  │  73% (2,847) │ │  27% (1,053)│   │
│  └─────────────┘ └─────────────┘   │
│                                     │
│  👨‍⚖️ AI 판결: A측 승 (87:42)       │
│  [내 의견 남기기] [판결문 보기]       │
└─────────────────────────────────────┘
```

**기대 효과**:
- 콘텐츠 소진 문제 해소 (매일 새 주제)
- 상대방 초대 병목 제거 (혼자도 참여 가능)
- 관전자(45%) 활성화
- SEO 유입 경로 확보 ("택배 분실 책임 누구" → Verdict 노출)

**구현 우선순위**: **P0 - MVP에 포함**

| 항목 | 스펙 |
|------|------|
| 주제 공급 | 운영팀 수동 3개/주 + AI 자동 생성 4개/주 |
| 참여 방식 | A/B 선택 → 짧은 의견(140자 이내) → AI가 양측 요약 후 판결 |
| 점수 반영 | AI 판결과 같은 측을 선택한 사용자에게 "통찰력 점수" +3 |
| 데이터 모델 | `daily_debates` 테이블 + `daily_votes` 테이블 추가 |

#### 3.2.2 커뮤니티 투표 (Verdict Vote)

완료된 판결문에 대해 "나도 A측 / B측 / AI가 틀렸다" 투표 기능.

```
┌─────────────────────────────────────┐
│  판결문 #3,847                       │
│  "결혼식 축의금 10만원은 적절한가"    │
│                                     │
│  AI 판결: A측 승 (적절하다)          │
│                                     │
│  🗳️ 시민 배심원단 투표:              │
│  A측 동의  ████████████░░░  78%     │
│  B측 동의  ███░░░░░░░░░░░░  18%     │
│  AI 오판   █░░░░░░░░░░░░░░   4%     │
│                                     │
│  참여: 4,203명                       │
└─────────────────────────────────────┘
```

**기대 효과**:
- 관전자의 능동적 참여 경로
- "AI가 맞는지 틀리는지" 확인하는 재미 → Hook Model의 Variable Reward
- 투표 데이터 축적 → AI 판결 품질 개선 피드백 루프 (Data Network Effect)
- 투표 정확도 기반 "시민 배심원 등급" 신설 가능

**구현 우선순위**: **P0 - MVP에 포함**

#### 3.2.3 논쟁 제안 게시판 (Topic Board)

사용자가 논쟁 주제를 제안하고, 다른 사용자가 "이거 궁금하다" 투표를 하면 인기 주제가 오늘의 논쟁으로 승격.

**기대 효과**:
- UGC 콘텐츠 엔진 가동 (사용자가 주제를 생산)
- 커뮤니티 참여감 → "내가 제안한 주제가 오늘의 논쟁이 되었다!"
- 콘텐츠 기획 비용 절감

**구현 우선순위**: P1 - 출시 1개월 내

### 3.3 Phase 2: 습관 형성 시스템 (출시 3~6개월)

> **핵심 원칙**: "Duolingo처럼 매일 돌아오게 만든다"

#### 3.3.1 판결 스트릭 (Verdict Streak)

```
🔥 연속 판결 참여: 14일

판결 스트릭 보상:
  3일 연속  → +5 보너스 점수
  7일 연속  → "주간 배심원" 배지 + 커스텀 프로필 테두리
  30일 연속 → "월간 판사" 배지 + 판결문 강조 표시
  100일+    → "법관" 칭호 + 특별 투표 가중치

  [스트릭 프리즈] 보유: 2개 (월 2개 자동 충전)
```

**핵심 설계 원칙** (Duolingo 학습):
- **스트릭 프리즈**: 하루 놓쳐도 스트릭이 깨지지 않는 아이템 (이탈 방지)
- **복귀 메커니즘**: 스트릭이 끊겨도 "연속 7일 복귀 챌린지"로 재참여 유도
- **가벼운 참여**: 직접 논쟁 제출이 아니라 "오늘의 논쟁 투표 1건"으로도 스트릭 유지

**데이터 모델 추가**:
```
user_streaks: {
  user_id, current_streak, longest_streak,
  streak_freezes_remaining, last_activity_date
}
```

#### 3.3.2 리그 시스템 (Weekly League)

Duolingo의 리그 시스템을 차용. 매주 월요일 리셋.

```
┌─────────────────────────────────────┐
│  ⚖️ 이번 주 리그: 실버 III          │
│  남은 시간: 2일 14시간               │
│                                     │
│  승격 구간 (상위 10명)               │
│  ─────────────────────              │
│  1. 논리왕김철수      847 XP  ⬆️     │
│  2. 판결의여왕        823 XP  ⬆️     │
│  3. ★ 나 ★           791 XP  ⬆️     │
│  ...                                │
│  ─────────────────────              │
│  유지 구간 (11~40명)                 │
│  41. ...              234 XP         │
│  ─────────────────────              │
│  강등 구간 (하위 10명)               │
│  42. ...              198 XP  ⬇️     │
└─────────────────────────────────────┘

리그 단계: 브론즈 → 실버 → 골드 → 플래티넘 → 다이아 → 대법관
```

**왜 현재 TOP 100 랭킹보다 나은가?**
- TOP 100은 상위 0.2%만 의미 있음 → 99.8% 사용자에게 무의미
- 리그는 50명 단위 소그룹 경쟁 → 모든 사용자가 승격/강등 긴장감 경험
- 매주 리셋 → 지속적 동기 부여 (전체 랭킹은 "따라잡기 불가능" 느낌)

#### 3.3.3 통찰력 프로필 (Insight Profile)

### 3.4 Phase 3: 네트워크 효과 구축 (출시 6~12개월)

> **핵심 원칙**: "Product에서 Platform으로 전환한다"

#### 3.4.1 소셜 그래프: 라이벌/팔로우

```
논쟁 상대 → 자동 라이벌 등록
  → "라이벌의 판결 활동" 피드
  → 라이벌과 직접 재대결 기능
  → "이번 주 라이벌 대결 성적: 3승 1패"
```

**네트워크 효과 유형**: Direct Network Effect (내 친구/라이벌이 있으므로 떠나기 어려움)

#### 3.4.2 콘텐츠 플라이휠 (Content Flywheel)

```
사용자가 논쟁 제출
     ↓
AI가 판결문 생성
     ↓
커뮤니티가 투표/토론  ←── 구경꾼 유입
     ↓
콘텐츠가 Google에 인덱싱
     ↓
"~은 누가 맞을까" 검색 → Verdict 노출
     ↓
신규 사용자 유입 → 논쟁 제출 or 투표
     ↓
(반복 - 플라이휠 가속)
```

**SEO 전략**:
- 모든 판결문을 공개 URL로 제공 (SSR + OG 태그)
- 카테고리별 인기 판결문 랜딩 페이지
- "~은 누구 잘못인가", "~이 맞나" 등 검색 패턴 타겟팅
- 장기적으로 판결문 아카이브가 곧 "논쟁 사전"이 됨

#### 3.4.3 커뮤니티 코멘트 (제한적 도입)

기존 계획에서 댓글을 Won't Have로 분류했으나, 영속성 관점에서 **제한적 코멘트 시스템** 도입 제안.

| 기존 우려 | 해결 방안 |
|-----------|-----------|
| 댓글이 논쟁 당사자를 공격할 수 있음 | 당사자가 아닌 "판결문"에 대한 의견으로 프레이밍 |
| 감정적 댓글로 커뮤니티 품질 저하 | AI 사전 필터링 + "논리적 의견만" 가이드라인 + 내공 기반 댓글 권한 |
| 핵심 가치(AI 판결) 희석 | 댓글은 판결 후에만 열림 → AI 판결이 항상 먼저 |

**구현**: 판결 완료 후 "시민 의견" 섹션에서 짧은 의견(200자) 작성 가능. 리그 실버 이상만 작성 가능 (스팸 방지).

### 3.5 Phase 4: 생태계 Lock-in (출시 12개월+)

> **핵심 원칙**: "떠나면 잃는 것이 너무 많게 만든다"

#### 3.5.1 축적형 자산 설계 (User Asset Accumulation)

사용자가 서비스를 사용할수록 쌓이는, 다른 곳으로 가져갈 수 없는 자산:

| 자산 유형 | 설명 | Lock-in 강도 |
|-----------|------|-------------|
| 판결 전적 | 승/패/무 기록, 카테고리별 통계 | ★★★ |
| 통찰력 프로필 | AI/시민 일치율, 분야별 정확도 | ★★★★ |
| 스트릭 기록 | 연속 참여 일수, 최장 기록 | ★★★★ |
| 리그 등급 | 브론즈~대법관 등급, 시즌 기록 | ★★★ |
| 칭호/배지 | 획득한 칭호, 시즌 한정 배지 | ★★★★★ |
| 제출 논쟁 아카이브 | 본인이 제출한 논쟁과 판결 결과 | ★★ |
| 라이벌 관계 | 대결 상대 기록, 승패 이력 | ★★★ |

#### 3.5.2 크리에이터 프로그램 (큐레이터/토너먼트 호스트)

파워 유저가 서비스 가치를 공동 창출하는 구조:

- **논쟁 큐레이터**: 좋은 논쟁 주제를 발굴/정리하는 사용자에게 보상
- **토너먼트 호스트**: 특정 주제("2026 연애 논쟁 토너먼트")를 기획/운영
- **판결 분석가**: 인기 판결에 심층 분석 콘텐츠를 작성하는 사용자

**Reddit 모델 참조**: 서브레딧 모더레이터 = 무급이지만 권한과 정체성이 보상. Verdict에서도 "운영"보다 "권한과 정체성"이 핵심 보상.

#### 3.5.3 데이터 네트워크 효과 (AI 판결 품질 해자)

```
더 많은 판결 → 더 많은 만족도 피드백 → 더 나은 프롬프트 튜닝
                                           ↓
경쟁사 진입 시:                        더 높은 판결 품질
"우리는 0건의 피드백 데이터"            ↓
vs Verdict "100만 건의 판결 + 피드백"  → 더 많은 사용자 유입
                                           ↓
                                      (경쟁사가 따라올 수 없는 격차)
```

이것은 Google 검색이 가진 것과 같은 종류의 해자(moat). 더 많은 사용을 통해 더 나은 품질을 만들고, 더 나은 품질이 더 많은 사용자를 끌어오는 선순환.

### 5.1 외부 트리거 → 내부 트리거 전환

```
[외부 트리거]                         [내부 트리거]
━━━━━━━━━━━━━━                      ━━━━━━━━━━━━━━
```

## 8. 핵심 지표 재설계: 영속성 관점

### 8.1 North Star Metric 재정의

```
기존: 월간 판결 완료 건수 (30,000건/월)
     → 문제: 판결은 이벤트성. "한 달에 몇 번 논쟁하는가?"는 통제 불가.

수정: 일간 활성 참여자 (Daily Active Participants)
     → 판결 + 투표 + 의견 + 열람 모두 포함
     → "매일 서비스에 관여하는 사용자 수"가 영속성의 진짜 척도
```

### 8.2 영속성 선행 지표
