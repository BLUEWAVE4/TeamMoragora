2 핵심 데이터 플로우
1. 논쟁 생성 플로우
1 Client: 3단계 위자드 (목적 → 렌즈 → 주제)
2 Axios POST /api/debates
3 Server: invite_code 생성 → Supabase INSERT
4 초대 링크 반환 → 클립보드 복사
2. AI 판결 생성 플로우
1 양측 주장 입력 완료 (50~2,000자)
2 Server: 3단계 콘텐츠 방어 통과 확인
3 Promise.allSettled → 3사 AI 병렬 호출
4 5항목×20점 루브릭 채점 + 판결문 JSON
5 verdicts + ai_judgments INSERT
3. 복합 판결 (AI+시민) 플로우
1 시민이 판결문 피드에서 A/B 투표
2 votes 테이블 INSERT (1인 1투표)
3 30명 이상 시 citizen 점수 산정
4 최종 = AI(60%) + 시민(40%) → verdicts UPDATE
4. 3단계 콘텐츠 방어 플로우
1 1단계: 비속어 사전 + 개인정보 정규식
2 2단계: GPT-4o AI 필터 (pass/warn/block)
3 3단계: AI 게이트키핑 (주제 적합성)
4 content_filter_logs 기록
3 기술 스택 상세
