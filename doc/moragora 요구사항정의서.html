Ⅰ 개요

1. 문서의 목적

본 문서는 "Moragora — AI 멀티모델 논쟁 판결 플랫폼" 개발의 요구사항을 정의한 문서이다.

상기 서비스는 두 사람의 논쟁 주제에 대해 4개 AI 모델(gpt-4.1, Gemini 2.5 Flash, Claude Sonnet 4, Grok-3-Mini(폴백))이 독립적으로 5항목×20점 루브릭 채점 및 판결을 내리고, 시민 투표와 결합한 복합 판결(AI 각 25% + 시민 25%)을 제공하는 것을 목표로 한다.

이 요구사항 정의서는 Moragora 웹서비스 개발을 위한 설계문서를 작성하는 데 기초가 되며, 사용자 유스케이스를 기반으로 SW가 제공해야 할 기능 및 화면에서 필수적으로 구현되어야 할 요구사항에 대해 기술하고 정의한다.

본 문서를 사용하는 대상은 본 과제를 기획하는 기획자, SW를 개발하는 개발자 등이며, 본 과제의 요구사항 도출 및 개발 과정에서 본 문서를 활용할 수 있도록 한다.

2. 요구사항 및 문서의 범위

본 문서에서는 유스케이스 및 기능·비기능 요구사항의 기술을 그 범위로 한다.

기능 범위: 핵심 기능(인증, 논쟁, AI 판결, 시민 투표, 콘텐츠 방어) + 실시간 채팅 논쟁 + 로비 3v3 아바타 + 알림 + 댓글 + 관리자 대시보드 + 오늘의 논쟁 + 회원탈퇴 + 소크라테스 코칭 + 합의/분석 판결모드를 포함하여 유스케이스 19건, 기능 요구사항 약 90건, 비기능 요구사항 12건을 정의합니다.
기술 스택: React 19 (Vite) + Express.js + Supabase (PostgreSQL + Auth + RLS) + Socket.io + AI 4사 API (판결: gpt-4.1, Gemini 2.5 Flash, Claude Sonnet 4, Grok-3-Mini(폴백) / 코칭: o4-mini / 주제분석: gpt-4.1-mini / Solo: gpt-4o-mini / 콘텐츠 필터: gpt-4o)
