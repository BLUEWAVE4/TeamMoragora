## 실험 문화 가이드라인

### 실험 원칙
1. 모든 주요 변경은 A/B 테스트로 검증 (직감 아닌 데이터 기반 의사결정)
2. 통계적 유의성(p < 0.05)과 비즈니스 유의성(실질적 임팩트) 모두 확인
3. 실험 결과는 성공/실패 관계없이 모두 기록 (실패에서도 학습)
4. 가드레일 지표 악화 시 즉시 실험 중단 (사용자 경험 보호)
5. 한 번에 너무 많은 변수를 변경하지 않음 (인과관계 명확화)
6. 최소 2주 이상 실험 기간 확보 (주말 효과, 노벨티 효과 배제)

### 실험 리뷰 프로세스
1. **가설 수립** → 팀 리뷰 (매주 월요일 Growth 미팅)
2. **실험 설계** → 통계 검증 (샘플 사이즈 계산기 활용)
3. **실행** → 중간 모니터링 (가드레일 지표 일간 체크)
4. **결과 분석** → 팀 공유 (매주 금요일 실험 리뷰)
5. **의사결정** → 적용/학습 기록 (이 문서에 즉시 반영)
