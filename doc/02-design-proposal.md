### 1.4 UX 트렌드 분석

1. **AI-Powered UX (AI 기반 사용자 경험)**: AI가 백그라운드 프로세서가 아닌 서비스의 전면에 등장하는 트렌드. Verdict에서는 3개 AI를 고유한 캐릭터로 의인화하여 "AI 판사"라는 메타포를 시각적으로 구현. 판결 과정을 투명하게 보여주는 Progressive Disclosure 적용.

2. **Gamification 2.0 (고도화된 게이미피케이션)**: 단순 포인트/뱃지를 넘어 의미 있는 진행감과 사회적 비교를 통한 동기 부여. 레이더 차트 기반 능력치 시각화, 티어 시스템, 연승 스트릭 등으로 재방문 동기 강화.

3. **Micro-Animation & Delight (마이크로 애니메이션과 즐거움)**: 결과 공개, 점수 변화, 랭킹 상승 등 핵심 순간에 "delight moment"를 연출. 판결 공개 시 법정 가벨 애니메이션, 점수 카운트업, 랭킹 변동 파티클 효과 등.

4. **Content as Interface (콘텐츠가 곧 인터페이스)**: 판결문 자체가 핵심 콘텐츠이자 인터페이스. 판결문의 가독성과 스캔 가능성을 극대화하는 타이포그래피 위계, 하이라이트, 접이식 상세 보기.

5. **Social Shareability (소셜 공유 최적화)**: OG 이미지 자동 생성, 공유 카드의 시각적 임팩트, 공유 후 유입 사용자의 전환 최적화. "나도 판결받기" CTA가 포함된 공유 카드 설계.

### 1.5 사용성 원칙 체크리스트 (Nielsen's 10 Heuristics)

- [x] **시스템 상태 가시성**: AI 판결 진행률(3단계 프로그레스), 상대방 입력 대기 상태, 초대 수락 여부 실시간 표시
- [x] **시스템과 실제 세계의 일치**: "법정/재판" 메타포 활용 (판결문, 변호, 가벨), 일상 언어 사용
- [x] **사용자 제어와 자유**: 논쟁 취소, 주장 수정(제출 전), "혼자 판결" 모드 전환 등 탈출구 제공
- [x] **일관성과 표준**: 디자인 토큰 기반 일관된 컴포넌트, 플랫폼 표준 패턴 준수
- [x] **오류 예방**: 글자 수 실시간 카운터, 부적절 콘텐츠 사전 필터링, 제출 전 확인
- [x] **기억보다 인식**: 논쟁 템플릿/예시 제공, 카테고리 시각적 아이콘, 최근 논쟁 히스토리
- [x] **유연성과 효율성**: 소셜 로그인 원클릭, 빠른 초대(링크 복사), 파워 유저를 위한 바로가기
- [x] **미적이고 미니멀한 디자인**: 법정 컨셉의 권위감을 유지하면서 불필요한 장식 배제, 핵심 정보 중심 레이아웃
- [x] **오류 인식/진단/복구 지원**: AI 응답 실패 시 구체적 안내 + 재시도 버튼, 입력 오류 하이라이트
- [x] **도움말과 문서**: 첫 사용 온보딩 튜토리얼, 주장 작성 가이드, FAQ 접근성

## 2. 정보 구조 설계 (Information Architecture)

### 2.1 사이트맵

```
Verdict (홈)
├── 홈 피드
│   ├── 인기 판결문 (트렌딩)
│   ├── 최신 판결문
│   ├── 카테고리별 판결문 (일상/연애/직장/사회/음식/문화/IT/기타)
│   └── 진행 중인 내 논쟁 상태 카드
├── 논쟁 생성
│   ├── 주제 입력 + 카테고리 선택
│   ├── 상대방 초대 (링크/카카오/QR)
│   ├── 주장 입력 (양측)
│   ├── 반박 라운드 (선택, 최대 2회)
│   └── 판결 요청
├── 판결 결과
│   ├── 판결 대기 화면 (프로그레스 + 추천 콘텐츠)
│   ├── 종합 판결 결과
│   ├── AI별 개별 판결 (GPT / Gemini / Claude)
│   ├── 점수 비교 (레이더 차트 + 항목별 상세)
│   ├── 만족도 평가
│   └── SNS 공유
├── 랭킹
│   ├── 전체 랭킹 (TOP 100)
│   ├── 주간 랭킹
│   ├── 월간 랭킹
│   └── 내 순위 & 주변 순위
├── 판결문 상세
│   ├── 논쟁 주제 & 양측 주장 요약
│   ├── 3개 AI 판결문 전문
│   ├── 점수 상세
│   ├── 좋아요/공감 반응
│   └── 공유 버튼
└── 마이페이지
    ├── 프로필 (닉네임, 전적, 총점, 랭킹, 티어)
    ├── 내 논쟁 히스토리 (진행중/완료/미완료)
    ├── 알림 설정
    ├── 계정 관리
    └── 고객 지원 / FAQ
```

### 2.2 네비게이션 구조

- **GNB (하단 탭 바, 모바일)**: 홈 | 논쟁 생성(+) | 랭킹 | 마이페이지
- **GNB (상단 내비게이션, 데스크톱)**: Logo | 홈 | 랭킹 | [논쟁 시작하기 CTA] | 프로필 아바타
- **LNB (로컬 내비게이션)**: 홈 피드 내 카테고리 탭, 랭킹 내 기간 탭, 판결 결과 내 AI별 탭
- **Depth**: 최대 3단계 (홈 > 판결문 상세 > AI별 판결 상세)

### 2.3 사용자 플로우 (주요 태스크)

#### 플로우 1: 논쟁 생성 ~ 판결 수령 (핵심 플로우)

```
[홈] → [논쟁 생성 탭] → [주제 입력 + 카테고리 선택]
  → [상대방 초대 (링크/카카오/QR)]
    → [상대방 수락 대기]
      ├─ (수락) → [양측 주장 입력] → [양측 입력 완료]
      │             → [판결 요청] → [판결 대기 (최대 3분)]
      │               → [판결 결과 확인] → [점수 확인]
      │                 → [만족도 평가] → [SNS 공유 / 홈 복귀]
      │
      ├─ (24시간 미수락) → [재초대 / 다른 상대 / 혼자 판결]
      │                     └─ (혼자 판결) → [단일 주장 AI 분석]
      │
      └─ (상대방 주장 미입력 48시간) → [재촉 / 취소 / 혼자 판결]
```

#### 플로우 2: 판결문 탐색 (관전자 플로우)

```
[SNS 공유 카드 클릭 / 홈 피드] → [판결문 상세]
  → [AI별 판결 탭 전환] → [좋아요/공감]
    ├─ (비회원) → [가입 유도 바텀시트] → [소셜 로그인] → [홈]
    └─ (회원) → [SNS 공유] / [나도 논쟁 시작하기]
```

#### 플로우 3: 랭킹 확인

```
[하단 탭: 랭킹] → [전체/주간/월간 탭 전환]
  → [TOP 100 리스트 스크롤] → [내 순위 하이라이트 확인]
    → [다른 플레이어 프로필 탭] → [해당 플레이어 전적 확인]
```

## 3. UX 설계 포인트

### 3.1 핵심 인터랙션 패턴

| 패턴 | 적용 화면 | 설명 |
|------|-----------|------|
| Step-by-Step Wizard | 논쟁 생성 | 주제 입력 > 카테고리 > 초대의 단계별 진행. 상단 프로그레스 바로 현재 위치 표시 |
| Progressive Disclosure | 판결 결과 | 종합 판결 > 개별 AI 판결 > 상세 점수 순으로 단계적 정보 공개. 탭/아코디언 활용 |
| Pull-to-Refresh | 홈 피드, 랭킹 | 당겨서 새로고침으로 최신 판결문/랭킹 갱신 |
| Infinite Scroll | 판결문 피드 | 카드 리스트의 무한 스크롤 로딩 (스켈레톤 UI 적용) |
| Bottom Sheet Modal | 공유, 초대, 가입 유도 | 모바일 최적화된 바텀시트로 컨텍스트 전환 최소화 |
| Swipe Navigation | AI별 판결 | 좌우 스와이프로 GPT/Gemini/Claude 판결 전환 (탭 + 스와이프 병행) |
| Drag Handle | 판결 대기 중 추천 카드 | 대기 화면에서 위로 스와이프하여 추천 판결문 카드 노출 |
| Floating Action Button | 논쟁 생성 | 모바일 우하단 FAB로 어디서든 빠른 논쟁 생성 접근 |

### 3.2 마이크로 인터랙션

| 트리거 | 피드백 | 목적 |
|--------|--------|------|
| 판결 요청 버튼 탭 | 가벨(법봉) 내려치는 애니메이션 + 진동(햅틱) | 판결 시작의 극적 순간 연출 |
| 판결 결과 공개 | 봉투 열리는 애니메이션 > 승/패 텍스트 스케일업 등장 | 결과 확인의 긴장감과 쾌감 |
| 점수 표시 | 0에서 최종 점수까지 카운트업 애니메이션 (이징: ease-out) | 점수의 체감 극대화 |
| 랭킹 상승 | 순위 숫자 슬롯머신 효과 + 상승 화살표 바운스 | 성취감 시각 피드백 |
| 좋아요 탭 | 하트 파티클 버스트 + 숫자 증가 애니메이션 | 즉각적 반응 피드백 |
| 글자 수 카운터 | 50자 미만: 빨간색 경고 흔들림, 200자 이상: 초록색 체크 전환 | 입력 가이드 실시간 피드백 |
| 상대방 입장 | 상대 아바타 슬라이드인 + "상대방이 입장했습니다" 토스트 | 실시간 상태 인지 |
| 공유 버튼 탭 | 카드 플립 애니메이션으로 공유 카드 프리뷰 전환 | 공유될 모습 사전 확인 |
| 탭 전환 (AI별) | 언더라인 인디케이터 슬라이드 + 컨텐츠 페이드 전환 | 부드러운 컨텍스트 전환 |
| 에러 발생 | 화면 미세 흔들림(shake) + 빨간 테두리 펄스 | 오류 즉각 인지 |

### 3.3 모바일 제스처 상세 스펙

#### 스와이프 제스처

| 제스처 | 적용 화면 | 임계값 (Threshold) | Velocity Threshold | 동작 |
|--------|-----------|-------------------|-------------------|------|

### 3.5 온보딩 플로우

```
[스플래시: 가벨 로고 애니메이션]
  → [웰컴: "논쟁의 마침표, AI가 찍어드립니다"]
    → [가치 1: "3개 AI가 독립적으로 판결합니다" + GPT/Gemini/Claude 캐릭터 소개]
      → [가치 2: "점수를 모아 랭킹 1위에 도전하세요" + 랭킹 미리보기]
        → [소셜 로그인: 카카오/구글/애플 원클릭]
          → [닉네임 설정 + 프로필 이미지 선택]
            → [첫 논쟁 유도: "첫 번째 논쟁을 시작해볼까요?" + 예시 논쟁 템플릿 3개 제공]
              → [홈 피드 (첫 사용자용 코칭 마크 오버레이)]
```

온보딩 핵심 원칙:
- 전체 온보딩은 3단계 이내, 30초 이내 완료
- 각 단계에서 "건너뛰기" 항상 제공
- 소셜 로그인 선택 즉시 서비스 진입 가능 (닉네임은 나중에 설정 가능)
- 첫 논쟁 유도에서 예시 템플릿 제공으로 진입 장벽 완화

    → "판결을 선고합니다" 텍스트 + 드럼롤 사운드(옵션)

[5] 상세 점수 공개
    → 양측 점수 바 동시 카운트업 (0 → 최종점수, 2초)

## 7. 추천 안 및 근거

### 추천 안: 안 A "Court of Fun" (위트 있는 법정)

### 선정 근거

1. **사용자 목표 정합성**: 세 페르소나 모두에 균형 있게 어필한다. 논쟁 해결자(김지훈)에게는 게이미피케이션 요소가, 구경꾼(이수연)에게는 위트 있는 시각적 재미가, 중재 활용자(박민수)에게는 법정 메타포의 신뢰감이 작동한다. 안 B(Neon Arena)는 Z세대에 편중되고, 안 C(Modern Court)는 게이미피케이션 감성이 약하다.

2. **비즈니스 목표 정합성**: SNS 바이럴이 핵심 성장 전략인데, 안 A의 "위트 있는 법정" 비주얼은 공유 카드의 시각적 차별성과 임팩트를 확보한다. 양피지 톤 카드에 가벨 아이콘, AI 캐릭터가 담긴 공유 카드는 SNS 피드에서 눈에 띈다. 법정 메타포가 서비스의 핵심 가치("AI 판결")와 가장 자연스럽게 연결되어 브랜드 인지도 형성에 유리하다.

3. **구현 용이성**: 안 B 대비 애니메이션/글로우 효과의 기술적 복잡도가 낮다. Pretendard 단일 폰트 기반으로 안 C의 세리프+산세리프 혼용 대비 관리가 단순하다. 일러스트 에셋은 초기 제작 비용이 있으나, 한 번 제작 후 재사용성이 높다. PWA 환경에서 안정적 렌더링이 보장된다.

4. **확장성**: 시맨틱 컬러 토큰 기반으로 다크 모드 추가가 용이하다. 법정 메타포는 향후 업적/뱃지 시스템(판사봉, 법복 등), 시즌 이벤트 등 확장 테마와 자연스럽게 연결된다. AI 캐릭터 시스템은 향후 AI 판결 성향 분석, 개인화 등 고도화 기능과 시너지를 낸다.

5. **접근성**: 밝은 Ivory 배경(#FAFAF5)과 Deep Black 텍스트(#1A1A2E)의 대비비 약 15:1로 WCAG AAA 수준 충족. AI 캐릭터 컬러(Green/Blue/Amber)도 색각이상자 고려한 조합이며, 아이콘 형태 차이로 컬러 외 구분 수단도 제공.

### 디자인 토큰 (개발 적용)

위 6.1절의 전체 토큰을 CSS Custom Properties로 구현한다. 핵심 사항은 다음과 같다.

- **컬러**: 시맨틱 토큰 사용 필수 (직접 HEX 금지). `--color-primary`, `--color-accent` 등
- **다크 모드**: `prefers-color-scheme: dark` 미디어 쿼리로 `--color-*-dark` 토큰 전환
- **AI 컬러**: `--color-judge-g`, `--color-judge-m`, `--color-judge-c`를 AI 관련 모든 컴포넌트에 일관 적용
- **폰트**: Pretendard Variable 사용 (woff2). `font-weight` 400/500/600/700/800 범위
- **간격**: 4px 기반 스페이싱 스케일 (--spacing-xs ~ --spacing-3xl)
- **전환**: 일반 인터랙션은 `--transition-normal` (250ms), 드라마틱 연출은 `--transition-dramatic` (800ms)

### 접근성 구현 체크리스트

- [ ] 모든 인터랙티브 요소에 `aria-label` 또는 의미 있는 텍스트 제공
- [ ] 색상만으로 정보를 전달하지 않음 (AI 구분: 컬러 + 아이콘 형태 + 텍스트 라벨 병행)
- [ ] 최소 터치 영역 44x44px (WCAG 2.1 AA)
- [ ] 텍스트 색상 대비비 4.5:1 이상 (본문), 3:1 이상 (대형 텍스트)
- [ ] 키보드 네비게이션 지원 (Tab/Enter/Escape)
- [ ] 스크린 리더 호환: 판결 결과, 점수, 랭킹 변동 등 동적 콘텐츠에 `aria-live` 적용
- [ ] 애니메이션 감소 모드 지원 (`prefers-reduced-motion: reduce` 대응)
- [ ] 폼 입력 필드에 명확한 라벨, 에러 메시지, 필수 표시

### 성능 고려사항

- 판결문 피드 무한 스크롤: 뷰포트 외 카드는 가상화(Virtualization) 적용 권장
- Lottie 애니메이션: JSON 에셋 경량화 (50KB 이하), 뷰포트 진입 시 로드
- 공유 카드 이미지 생성: 서버사이드 렌더링(html-to-image 또는 서버 Canvas) 권장
- 스켈레톤 로딩: 실제 레이아웃과 동일한 형태로 CLS(Cumulative Layout Shift) 최소화
- 폰트: Pretendard Variable woff2 단일 파일로 서브셋 적용 (한글 + 영문 + 숫자)
