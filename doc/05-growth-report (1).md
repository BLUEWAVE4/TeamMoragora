**인사이트**: Verdict의 가장 강력한 성장 엔진은 **"논쟁 생성 → 상대방 초대 → 판결문 공유"**라는 자연 바이럴 루프. 유료 광고 의존도를 최소화하고 바이럴 계수(K-factor)를 높이는 것이 핵심.

**액션 아이템**:
1. 공유 카드 시각 임팩트 최적화 (카카오톡/인스타그램 플랫폼별 최적 사이즈)

3. 초대 성공 시 추가 보너스 점수 부여 (초대자 인센티브)

### 2.2 Activation (활성화)

#### 핵심 활성화 퍼널 정의
| 단계 | 행동 | 목표 전환율 | 이탈 방지 전략 |
|------|------|------------|---------------|
| Step 1: 앱 방문 | 랜딩 페이지 도착 | 100% (기준) | - |
| Step 2: 가입 완료 | 소셜 로그인 원클릭 | 30% | 소셜 로그인으로 가입 허들 최소화, 게스트 모드 고려 |
| Step 3: 첫 논쟁 생성 | 논쟁 방 개설 | 60% (가입자 대비) | 온보딩 튜토리얼, 인기 주제 추천, "이 주제로 바로 시작" 버튼 |
| Step 4: 상대방 초대 성공 | 초대 수락 | 40% (생성자 대비) | 다중 초대 수단(링크/카카오/QR), 미수락 시 "혼자 판결" 전환 |
| Step 5: **첫 판결 수령** (Aha Moment) | AI 판결문 열람 | 85% (양측 입력 대비) | 판결 대기 이탈 방지(카운트다운, 추천 콘텐츠), 알림 |
| Step 6: 판결 공유 | SNS 공유 클릭 | 25% (판결 수령 대비) | 공유 카드 시각적 임팩트, 원클릭 공유, 공유 인센티브 |

**Aha Moment**: **"첫 AI 판결문을 받아보는 순간"** - 3개 AI가 각각 다른 시각에서 논리적 판결을 내리고, 승패와 점수가 결정되는 경험이 서비스의 핵심 가치 체험 순간.

**인사이트**: Aha Moment 도달률(가입 → 첫 판결 수령)이 전체 퍼널의 핵심. 특히 "상대방 초대 성공" 단계(40%)가 가장 큰 병목으로 예상됨. "혼자 판결" 모드의 빠른 구현이 활성화율을 크게 개선할 수 있음.

**액션 아이템**:
1. "혼자 판결" 모드 v1.0 포함 (초대 미수락 시 퍼널 이탈 방지)
2. 온보딩 시 "인기 논쟁 주제 TOP 10" 제안으로 첫 논쟁 생성 유도
3. Aha Moment 도달 소요 시간 모니터링 (목표: 가입 후 10분 이내)

### 2.3 Retention (유지)

#### 리텐션 전략 프레임워크
| 시점 | 유지 동기 | 전략 | 목표 유지율 |
|------|-----------|------|------------|
| D1 | 판결 결과 확인 + 반박 | 판결 완료 푸시 알림, 반박 라운드 유도 | 40% |
| D3 | 새로운 논쟁 시작 | 인기 주제 추천 알림, "오늘의 논쟁" | 30% |
| D7 | 랭킹 경쟁 | 주간 랭킹 업데이트 알림, 랭킹 상승 축하 | 20% |
| D14 | 콘텐츠 소비 | 인기 판결문 피드 업데이트, 관심 카테고리 알림 | 15% |
| D30 | 습관 형성 | 월간 랭킹, 월간 통계 리포트 ("이번 달 전적"), 연승 스트릭 | 10% |

#### 핵심 유지 루프
```
사용자 A: 논쟁 생성 → 판결 수령 → 점수 획득 → 랭킹 확인 → 새 논쟁 생성
                                           ↓
                                    판결문 공유 → 다른 사용자 유입
                                           ↓
사용자 B (관전자): 피드 열람 → 흥미 → 가입 → 논쟁 생성
```

**리텐션 커브 형태 예측**: 초기 Decline → D7~D14에서 안정화 예상 (Flat). 게이미피케이션(점수/랭킹/연승)이 D7 이후 유지에 핵심.

**인사이트**: D1 유지율이 가장 중요. 첫 판결 후 "다음 논쟁"을 유도하는 경험 설계가 D1→D7 전환의 핵심. 랭킹이 의미를 갖기 위해서는 최소 3회 이상의 판결 경험이 필요하므로, 초기 3회 판결 완료까지의 퍼널 집중 관리가 필요.

**액션 아이템**:
1. "3회 판결 완료" 마일스톤 보상 (뱃지 + 보너스 점수)
2. D1 알림: "당신의 판결 결과가 N명에게 열람되었습니다" (사회적 증거)
3. D7 알림: "이번 주 랭킹이 N위 상승했습니다!" (경쟁 동기)
4. 월간 개인 리포트: "이번 달 N회 판결, 승률 M%, 랭킹 K위" (성취 가시화)

### 2.5 Referral (추천)

#### 바이럴 루프 분석
| 지표 | 정의 | 목표 |
|------|------|------|
| K-factor (바이럴 계수) | 1명이 평균 몇 명을 데려오는가 | 0.5 이상 (3개월), 0.8 이상 (6개월) |
| 초대 발송률 | 논쟁 생성자 중 초대 발송 비율 | 90% (논쟁 구조상 자연 발생) |
| 초대 수락률 | 초대 받은 사람 중 수락 비율 | 40% (3개월), 50% (6개월) |
| 공유 → 가입 전환 | 공유 카드 본 사람 중 가입 비율 | 5% (3개월), 8% (6개월) |

#### 바이럴 루프 종류
1. **필수 바이럴** (논쟁 초대): 서비스 사용 구조상 반드시 상대방 초대 필요 → K-factor 기본값
2. **자발적 바이럴** (판결문 공유): 재미있는 판결 결과를 SNS에 공유 → 추가 K-factor
3. **관전 바이럴** (피드 열람): 공개 판결문 피드 → "나도 해보고 싶다" → 가입

**인사이트**: Verdict는 구조적으로 바이럴이 내장된 서비스(논쟁에 2명 필요). 이 자연 바이럴에 판결문 공유 바이럴이 더해지면 K-factor > 1.0도 가능.

**액션 아이템**:
1. 초대 수락률 최적화: 초대 메시지 문구 A/B 테스트, 카카오톡 공유 카드 디자인 최적화
2. 공유 카드에 "나도 판결받기" CTA 강화 + 공유한 사람의 닉네임/점수 표시 (사회적 동기)
3. 공유 보상: 공유 후 열람 수에 따른 보너스 점수 (공유 인센티브)

## 5. Pre-Launch 하이라이트

### 하이라이트 (강점)
1. **구조적 바이럴**: 논쟁 특성상 2인 초대가 필수 → 자연 바이럴 내장
2. **콘텐츠 자동 생성**: AI 판결문이 곧 콘텐츠 → 공유 가능한 소셜 자산
3. **게이미피케이션 리텐션**: 점수/랭킹/뱃지가 재방문 동기 제공
4. **차별화된 UX**: 3개 AI 교차 판결은 경쟁 서비스에 없는 독보적 경험
5. **SEO 친화적**: 공개 판결문 피드가 검색 유입 채널로 작동

### 리스크 (로우라이트)
1. **AI 비용 관리**: MAU 증가에 비례하여 AI 비용 선형 증가 → 수익화 전까지 현금 소모
2. **초대 병목**: 상대방 초대 실패 시 퍼널 이탈 → "혼자 판결" 미지원 시 치명적
3. **콘텐츠 품질**: AI 판결의 논리적 품질이 사용자 신뢰와 직결 → 프롬프트 최적화 필수
4. **법적 리스크**: AI 판결의 오용 가능성 (실제 법적 분쟁에 인용) → 면책 고지 강화

### Pre-Launch 학습
1. 유사 서비스(밸런스 게임, ChatGPT 직접 질문)의 한계가 Verdict의 기회
2. 바이럴 계수 K-factor > 0.5이면 유료 마케팅 의존도를 크게 낮출 수 있음
3. 판결 만족도 4.0 이상 유지가 장기 리텐션의 전제 조건

## 6. 출시 후 포커스

### 최우선 과제 (Top 3)
1. **Aha Moment 도달률 최적화**: 가입 → 첫 판결 수령 퍼널 전환율 → 목표 15%+(방문자 대비)
2. **바이럴 계수 측정 및 최적화**: K-factor 측정 시작, 공유 카드 CTA A/B 테스트 → 목표 K > 0.5
3. **D7 유지율 20% 달성**: 3회 판결 마일스톤 + 주간 랭킹 알림 → D7 유지율 모니터링

### 실험 계획
- **EXP-001**: 초대 메시지 문구 A/B 테스트 (초대 수락률 40% → 50%)
- **EXP-002**: 공유 카드 디자인 변형 (공유 → 가입 전환율 5% → 8%)
- **EXP-003**: 온보딩 플로우 간소화 (가입 → 첫 논쟁 생성 전환율 60% → 70%)
- **EXP-004**: "혼자 판결" 모드 도입 효과 (초대 병목 해소 → 전체 활성화율)
